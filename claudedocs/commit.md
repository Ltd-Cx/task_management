# Backlog Clone - Pencil デザイン仕様プロンプト

shadcn/ui (new-york style) でデザインしてください。
全画面共通のレイアウトとして、左にサイドバー、右にメインコンテンツを配置します。

---

## 共通レイアウト

### サイドバー（全画面共通）
- 上部ブランド: 「Backlog Clone」
- プロジェクト名: 「サンプルプロジェクト」
- ナビゲーションリスト:
  1. ダッシュボード ← 課題一覧画面でactive
  2. 課題 ← 課題一覧画面でactive
  3. ボード ← ボード画面でactive
  4. ガントチャート ← ガント画面でactive
  5. メンバー ← メンバー画面でactive
  6. プロジェクト設定 ← 設定画面でactive

### ヘッダー（全画面共通）
- サイドバートグルボタン
- セパレーター
- パンくずリスト: 「サンプルプロジェクト」 > 「{現在のページ名}」

---

## Screen 1: 課題一覧

**パンくず**: サンプルプロジェクト > 課題

### ツールバー
- 左: タイトル「課題」
- 右: 「課題を追加」ボタン

### データテーブル

**テーブルヘッダー（7列）:**
| 列名 | 内容 |
|------|------|
| キー | プロジェクトキー-連番（例: SAMPLE-1） |
| 件名 | 課題のタイトルテキスト |
| 状態 | ステータスBadge（ドット付き） |
| 優先度 | 優先度Badge |
| 担当者 | アバター + 名前 |
| カテゴリー | カテゴリーBadge（outline） |
| 更新日 | 日付テキスト（YYYY/MM/DD） |

**サンプルデータ（3行）:**
1. SAMPLE-1 | ログイン機能の実装 | 処理中 | 高 | 田中太郎 | フロントエンド | 2026/02/20
2. SAMPLE-2 | ダッシュボードデザイン | 未対応 | 中 | 鈴木花子 | デザイン | 2026/02/19
3. SAMPLE-3 | API設計書の作成 | 完了 | 低 | 佐藤一郎 | バックエンド | 2026/02/18

## Screen 2: 課題追加/編集ダイアログ

**背景**: Screen 1（課題一覧）をオーバーレイ暗転した状態

### ダイアログ
**ヘッダー**: 「課題の追加」

**フォーム内容（上から順に）:**

1. **件名**（必須） - Input、placeholder: 「課題のタイトルを入力」
2. **詳細** - Textarea、placeholder: 「課題の詳細を入力（任意）」
3. **状態 + 優先度**（2列グリッド）
   - 状態 Select（未対応/処理中/処理済み/完了）
   - 優先度 Select（高/中/低）
4. **担当者 + カテゴリー**（2列グリッド）
   - 担当者 Select（placeholder: 「担当者を選択」）
   - カテゴリー Select（placeholder: 「カテゴリーを選択」）
5. **開始日 + 期限日**（2列グリッド）
   - 開始日 date Input
   - 期限日 date Input

**フッター**: 「キャンセル」ボタン（outline）+「追加する」ボタン（primary）

## Screen 3: ボード（カンバンビュー）

**パンくず**: サンプルプロジェクト > ボード

### ツールバー
- 左: タイトル「ボード」

### カンバンレイアウト
横並び4列。各列が独立して縦スクロールする。

**4つのステータス列（各列ヘッダーにドット + ラベル + 件数Badge）:**
1. 未対応
2. 処理中
3. 処理済み
4. 完了

### タスクカード（各列内）
- タスクキー（例: SAMPLE-1）
- 件名
- 下段: 優先度Badge + アバター

**サンプルデータ:**
- 未対応列: 2件（SAMPLE-2, SAMPLE-5）
- 処理中列: 2件（SAMPLE-1, SAMPLE-4）
- 処理済み列: 1件（SAMPLE-6）
- 完了列: 1件（SAMPLE-3）

## Screen 4: ガントチャート

**パンくず**: サンプルプロジェクト > ガントチャート

### ツールバー
- 左: タイトル「ガントチャート」
- 右: 「<」ボタン +「今日」ボタン +「>」ボタン

### メインエリア（横2分割）

#### 左パネル: タスクリスト
- ヘッダー: 「課題」ラベル
- タスク行: タスクキー + 件名

**サンプルデータ（6行）:**
1. SAMPLE-1 | ログイン機能の実装
2. SAMPLE-2 | ダッシュボードデザイン
3. SAMPLE-3 | API設計書の作成
4. SAMPLE-4 | テスト環境構築
5. SAMPLE-5 | ユーザー管理画面
6. SAMPLE-6 | パフォーマンス改善

#### 右パネル: タイムライン（横スクロール可能）

- **月ヘッダー**: 「2026/2月」「2026/3月」
- **日付ヘッダー**: 日付 + 曜日（土日は色分け）
- **ガントバー行**: 背景グリッド（各日に縦線、週末薄色）+ ガントバー（ステータス色、バー内に白文字で件名）+ 左右リサイズハンドル

2. SAMPLE-2: 2/20-3/5（未対応）

## Screen 6: プロジェクト設定

**パンくず**: サンプルプロジェクト > プロジェクト設定

### ツールバー
- 左: タイトル「プロジェクト設定」

### 3つのカードセクション（縦並び）

#### Card 1: 基本設定
1. プロジェクト名 - Input（値: 「サンプルプロジェクト」）
2. プロジェクトキー - Input（値: 「SAMPLE」、disabled）、説明: 「プロジェクトキーは変更できません」
3. 説明 - Textarea
4. 「キャンセル」ボタン +「保存」ボタン

#### Card 2: カスタムステータス
ヘッダー: タイトル「カスタムステータス」+「追加」ボタン

説明: 「デフォルトステータス（未対応・処理中・処理済み・完了）に加えて、プロジェクト固有のステータスを追加できます。」

ステータスリスト（各行）:
- ドラッグハンドル + カラードット + 表示名 + キー + 編集ボタン + 削除ボタン

**サンプルデータ（4行）:**
1. 未対応 | open
2. 処理中 | in_progress
3. 処理済み | resolved
4. 完了 | closed

#### Card 3: 危険な操作（赤ボーダー）
- タイトル: 「危険な操作」
- 説明: 「プロジェクトを削除する」「この操作は元に戻すことができません。すべてのデータが削除されます。」
- 「削除」ボタン（destructive）
